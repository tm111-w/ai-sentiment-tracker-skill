# 舆情报告内容大纲

本文件定义了报告的内容结构。最终输出应为一个自包含的 HTML 页面，而非 .md 文件。

## 报告元信息
- 产品名称
- 报告生成时间
- 数据抓取范围（时间段）
- 数据来源平台列表

## 概览摘要（顶部卡片区域）
展示为视觉突出的摘要卡片：
- 总收集数据量
- 用户实测案例数
- 技术讨论数
- 功能好评数
- 功能差评/问题数
- KOL观点数
- 整体情感分布（正面/中性/负面百分比）
- 平台分布

## 用户实测案例精选 (Good Cases)
按质量排序，每个案例包含：
- 来源平台、作者、日期
- 任务描述
- 执行结果（成功/部分成功/失败）
- 用户评价摘要
- 原文链接

## 技术讨论汇总
按讨论热度排序，每个话题包含：
- 技术主题
- 讨论热度
- 核心观点列表
- 代表性帖子链接

## 功能好评精选
- 表格形式展示：用户、功能点、评价摘要、链接
- 最受好评的功能点排行

## 功能差评与问题
- 表格形式展示：问题类型、频次、典型描述、代表性反馈链接
- 用户反馈的主要痛点汇总

## KOL观点汇总
每位KOL包含：
- 姓名、身份背景
- 核心观点摘要（200字以内）
- 情感倾向
- 原文链接

## 竞品对比讨论
用户讨论中提及的竞品对比：竞品名称、对比维度、用户结论

## 分析洞察
- 产品优势（用户认可的点）
- 改进建议（用户期望的点）
- 市场机会
- 潜在风险

## 附录
- 数据来源明细（搜索执行次数、各平台搜索次数、有效结果数）
- 搜索关键词列表
- 数据限制说明

## HTML 输出格式要求

生成 HTML 报告时遵循以下设计规范：

### 视觉设计
- 使用现代化 CSS：渐变色背景、卡片阴影、圆角边框
- 概览摘要使用彩色卡片网格（正面绿色、负面红色、中性灰色）
- 表格使用斑马纹样式，hover 高亮当前行
- 链接使用产品品牌色或蓝色，可点击跳转

### 数据可视化
- 情感分布：饼图或环形图（推荐使用 Chart.js CDN，或纯 CSS/SVG）
- 平台分布：横向柱状图
- 功能好评排行：水平条形图

### 技术要求
- 单文件自包含（内联 CSS 和 JS）
- 外部依赖仅允许 CDN 引用（如 Chart.js）
- 响应式布局（移动端友好）
- 使用 UTF-8 编码

### 配色方案建议
- 主色调：深蓝 #1a1a2e 或科技感渐变
- 正面情感：#10b981（绿色）
- 负面情感：#ef4444（红色）
- 中性情感：#6b7280（灰色）
- 卡片背景：#ffffff，阴影 rgba(0,0,0,0.1)
